# MainWindow UI 重设计方案

## 目标

用项目的 `ComponentLibrary`（Ant Design 5 风格）重构 `MainWindow` 布局，替换当前纯 ImGui 平铺 Tab 结构，提升美观度和可用性。

## 模块划分（左侧 3 张固定卡片）

| 卡片 | 子 Tab |
|------|--------|
| **主控** | `[状态]` `[设置]` `[窗口设置]` |
| **调试** | `[Debug]` `[ACR Debug]` `[日志]` |
| **时间轴** | `[录制]` `[事实轴]` `[执行轴]` `[辅助轴]` |

> 注：`日志` 从原有「编辑器」模块调整至「调试」模块。

## 区域详解

### 1. 顶部信息栏

```
┌─ Logo ────────────┬─────── Tips 轮播区 ───────────┬── 控件 ──┐
│                   │  💡 <3~5条提示自动轮播，     │           │
│   HiAuRo          │      每条显示 3~4 秒后切换>   │ 🌙  ─ □ × │
│   ASCII Art       │                               │           │
└───────────────────┴───────────────────────────────┴───────────┘
```

- **Logo（左上）**：ASCII Art "HiAuRo" 文字，跟随主题色，固定 5~6 行高
- **Tips（中部）**：水平滚动/淡入淡出轮播，展示内容如：
  - 新功能简介
  - 快捷键提示
  - 版本更新动态
  - 纯展示，非交互
- **控件（最右）**：
  - 🌙 — 主题切换（Light/Dark），使用 `ComponentLibrary.IconButton`
  - ─ □ × — 窗口最小化/最大化/关闭（标准 Dalamud Window 控件）

### 2. 左侧栏卡片区

#### 固定模块卡片（极简风格）

```
未选中状态                    选中状态
┌──────────────────────┐    ┌──────────────────────┐
│ ▌  ◉  主控           │    │ ▋  ▣  主控           │
│                    │    │    当前模块          │
└──────────────────────┘    └──────────────────────┘
```

- 宽度 ~160px，高度 ~44px
- 左侧 3px 竖条：未选中灰色 / 选中主题蓝色
- 图标：未选中暗色 / 选中高亮
- 圆角 6px（`Theme.RadiusMD`），间距 6px

#### Plugin 动态卡片

```
固定模块卡片
───────────── ← 分隔线 (ComponentLibrary.Divider)
┌──────────────────────┐
│ ▌  ◉  PluginNameA   │
│       v1.2.0  · 🟢   │
└──────────────────────┘
┌──────────────────────┐
│ ▌  ◉  PluginNameB   │
│       v0.3.1  · 🔴   │
└──────────────────────┘
```

- **来源**：扫描 `Plugins/*.dll`，发现 `IPlugin` 实现即自动生成卡片
- **状态灯**（ComponentLibrary.Badge）：🟢 运行中 / 🟡 初始化 / 🔴 异常 / ⚪ 未加载
- **交互**：
  - 左键点击 → 嵌入内容区（调用 `IPlugin.GetEmbeddedUI()`）
  - 右键菜单 → 弹出独立窗口（调用 `IPlugin.GetWindow()`，保持现有行为）
- **排序**：固定模块始终在上，Plugin 按名称字母排序
- **空态**：无 Plugin 时隐藏分隔线和区域

### 3. Tab 栏

使用 `ComponentLibrary.Tabs`，水平排列在内容区顶部。Tab 列表跟随左侧卡片选中状态联动切换。

### 4. 内容区

显示当前模块 + 当前 Tab 的原有内容。保持各 `DrawXxx()` 方法不变，仅调整渲染容器。

Plugin 卡选中时替换内容区为该 Plugin 的面板。

### 5. 底部状态栏

```
HiAuRo v1.0.0  │  [就绪]  │  ACR: BRD(已加载)  │  Plugins: 2/2
```

- 灰色小字，背景略暗于内容区
- 显示：版本号、运行状态、ACR 状态、Plugin 统计

## ComponentLibrary 组件使用

| 组件 | 用途 |
|------|------|
| `GlassBackground` | 整体窗口半透明毛玻璃底色 |
| `Card` | 模块卡片 + Plugin 卡片容器 |
| `Tabs` | 子 Tab 导航栏 |
| `Badge` | Plugin 状态指示灯 |
| `Divider` | 固定模块与 Plugin 之间的分隔线 |
| `IconButton` | 主题切换按钮 |
| `Theme.Colors` | 全局主题色、选中态高亮 |

## Plugin 接口扩展

为支持嵌入内容区，`IPlugin` 接口需新增可选方法：

```csharp
namespace HiAuRo;

public interface IPlugin : IDisposable
{
    string Name { get; }
    string Version { get; }
    void Initialize();
    void Update();
    IPluginWindow? GetWindow() => null;

    // 新增：返回嵌入主窗口的 UI 绘制委托，与独立窗口并存
    Action? GetEmbeddedUI() => null;
}
```

- `GetWindow()` → 右键弹出独立窗口（保持现有行为）
- `GetEmbeddedUI()` → 嵌入内容区的 UI（新功能）
- 返回 `null` 表示不嵌入

## 实施范围

1. **MainWindow.cs** — 重写 `Draw()` 布局结构，引入卡片+Tab 联动框架
2. **IPlugin.cs** — 新增 `GetEmbeddedUI()` 方法
3. **ComponentLibrary.cs** — 评估是否需要新增组件（如轮播 Tips、侧边卡片列表等）

## 约束

- 不修改现有 `DrawXxx()` 方法的内部逻辑
- 不修改 `PluginWindowManager` 现有行为
- 不修改 ACR/ACR Debug 等已有 Tab 的功能
- 使用项目已有的 `ComponentLibrary` + `Theme` 体系，不引入新依赖
