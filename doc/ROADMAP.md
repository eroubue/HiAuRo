# HiAuRo — 路线图

## 当前状态

HiAuRo v1 核心框架已全部完成，包含以下模块：

| 模块 | 说明 | 状态 |
|------|------|------|
| 宿主层 | 插件生命周期、DService 接入 | ✅ |
| 基础设施 | 配置、日志、调试开关 | ✅ |
| 数据层 | HiAuRo.Data 统一入口（Self/Target/Party/Objects/Combat） | ✅ |
| 运行时核心 | 每帧 Tick、战斗上下文、ACR 生命周期、模式切换 | ✅ |
| ACR 抽象 | IRotationEntry/SlotResolver/Opener/Sequence/Trigger/Hotkey/QT/UI | ✅ |
| 执行轴 | 条件驱动执行控制（async AST + WaitCond + Roslyn 脚本） | ✅ |
| 事实轴 | Boss 时间线 JSON 建模（阶段 + Sync 校准 + 分支切换） | ✅ |
| 决策层 | 消费事实轴 + 贪心分配减伤/治疗 | ✅ |
| 创作工具 | 可视化编辑器（执行轴/事实轴/辅助轴） | ✅ |
| HiAuRo.Helper | 21 职业辅助库（独立仓库，自动更新） | ✅ |

---

## 全局规则

- 代码简单直接，不过度架构化
- 优先直接使用已有 API，不新增包装层
- 中文注释
- 新能力以 additive 方式叠加，不做推翻式重构
- ACR 开发模式和主要接口保持稳定，降低迁移成本
