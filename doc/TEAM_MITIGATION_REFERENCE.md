# FFXIV 团队减伤/治疗技能 StatusID 参考表
# (Dawntrail 7.0 - DT)

> 基于项目 DecisionEngine.cs 已有数据 + FFXIV 公开数据库资料汇编。
> **StatusID 是最重要的字段** — 用于程序化检查 buff/debuff 是否正在生效。
> 部分 StatusID 需要通过 xivapi.com 或游戏实测二次验证。

---

## 说明

- **状态ID**: Buff/Debuff 应用到目标上的 Status ID。团队减伤通常检查队友身上的 buff，雪仇等检查敌方的 debuff。
- **减伤类型**: 全能(物理+魔法) / 物理 / 魔法
- **已有**: ✅ = 已在 DecisionEngine.cs 中注册，❓ = 需验证，🆕 = 待补充

## 七、决策引擎使用说明

已注册的技能可在 DecisionEngine.cs 中找到。
注册新技能使用:

```csharp
DecisionSkillRegistry.注册(Jobs.XXX,
    teamMit: [ new() { 技能ID = ..., 名称 = "...", 状态ID = ..., ... } ],
    personalMit: [ ... ],
    teamHeal: [ ... ]);
```

StatusID 用途:
- `AuraHelper.HasAura(character, statusId)` - 检查角色是否有指定 buff
- `扫描已有减伤()` 方法遍历所有已注册的 StatusID，检查队伍成员和敌方目标
