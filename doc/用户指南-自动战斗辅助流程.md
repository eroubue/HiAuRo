# HiAuRo 自动战斗辅助 — 它是怎么帮你打副本的？

## 一句话总结

HiAuRo 就像一个经验丰富的副驾驶——你负责跑位和处理机制，它帮你按技能。它知道 Boss 什么时候放什么技能，知道你现在该用什么技能，每秒帮你做出几十次决策。

## 第一步：进入副本，系统自动准备

你传送到副本里的那一刻，HiAuRo 就开始工作了：

1. **识别你的职业** — 看你现在玩的是诗人、白魔法师还是黑魔法师
2. **加载战斗方案** — 每个职业都有专门的"战斗方案"（由职业作者写好的技能优先级逻辑）
3. **检查有没有预设的时间表** — 如果这个副本有提前准备好的攻略数据，系统会自动加载

> 💡 战斗方案是独立的小插件，可以单独更新。你不需要手动配置任何东西。

---

## 第二步：倒计时，准备起手

副本里出现"距离战斗开始还有 X 秒"的倒计时时：

- 系统会按照提前设定好的"起手序列"排队
- 比如倒计时 5 秒时准备一个增益技能，倒计时 2 秒时准备一个攻击技能
- 倒计时结束的瞬间，第一个技能立刻打出去，起手不浪费一毫秒

---

## 第三步：战斗开始，进入智能循环

战斗正式开始后，HiAuRo 每秒会帮你做几十次决策。每次决策的流程是：

```
┌─────────────────────────────────────────────────────────┐
│                    每一帧的决策过程                        │
│                                                          │
│  ①  刷新周围信息                                         │
│      → 哪些敌人在附近？队友状态如何？                      │
│                                                          │
│  ②  选择目标                                             │
│      → 如果没有目标，自动选一个最合适的敌人                 │
│                                                          │
│  ③  听取"指挥官"的意见（如果有）                          │
│      → 执行轴/事实轴可能会说："现在用这个技能！"           │
│      → 或者说："现在停手！"                               │
│                                                          │
│  ④  自己做决策                                           │
│      → 遍历所有可用的技能                                 │
│      → 检查条件：冷却好了吗？资源够吗？能打到目标吗？      │
│      → 按优先级选出最佳技能                               │
│                                                          │
│  ⑤  释放技能                                             │
│      → 告诉游戏："我要对这个目标使用这个技能"              │
└─────────────────────────────────────────────────────────┘
```

这个过程每秒重复几十次，你感觉不到任何延迟。

## 三种战斗模式

HiAuRo 有三种战斗模式，根据副本内容自动选择：

### 模式 A：无轴模式（基础自动战斗）

```
你 → HiAuRo 选技能 → 释放
```

最简单的模式。没有额外的"指挥官"，HiAuRo 只根据当前情况（技能冷却、目标血量、自身状态）自动选择最优技能。

适合：没有预设数据的副本、日常刷本。

---

### 模式 B：执行轴模式（按剧本打）

```
剧本（执行轴）→ 指挥 HiAuRo → 释放技能
```

系统加载一份提前写好的"战斗剧本"，里面写着：

- "战斗开始第 3 秒 → 用爆发技能"
- "Boss 读条某个技能时 → 切换到 AOE 模式"
- "Boss 血量低于 30% → 切换到全力输出模式"
- "某个阶段 → 停手，不要打"

执行轴就像一个严格的导演，按时间表指挥你的战斗。

适合：高难度副本、需要精确时间轴配合的场景。

---

### 模式 C：事实轴模式（智能模式，最强大）

```
Boss 时间线 → 理解当前局势 → 智能分配减伤/治疗 → 释放技能
```

这是最智能的模式，分三层工作：

**第一层：看懂 Boss（事实轴）**

系统有一份 Boss 的"技能时间表"，知道：
- 现在是第几阶段
- Boss 下一个技能是什么
- 还有多久 Boss 会放全屏 AOE

**第二层：做出决策（决策引擎）**

根据 Boss 即将放的技能，自动决定：
- 需要多少减伤？
- 谁来放减伤？（看队伍里有哪些职业，谁的减伤技能好了）
- 需要多少治疗？

比如 Boss 马上要放一个全屏大招，系统会：
1. 算出队伍需要 20% 的减伤
2. 发现骑士的"壁垒"好了（冷却 90 秒，减伤 15%）
3. 发现学者的"鼓舞"好了（冷却 60 秒，减伤 10%）
4. 自动分配：骑士放壁垒 + 学者放鼓舞 = 25% 减伤，够了！

**第三层：执行（ACR）**

把决策结果交给职业战斗方案去执行。

适合：零式/绝级副本、需要团队配合的高难度内容。

---

## 辅助轴：全程守护的隐形助手

不管你用的是哪种模式，"辅助轴"始终在后台运行。

它负责一些通用的、不需要复杂决策的小任务，比如：
- 特定副本的特殊机制提醒
- 简单的条件触发动作

它从一个独立的配置文件加载，不影响主战斗逻辑。

## 常见问题

**Q: 我需要手动配置什么吗？**

不需要。安装插件后，系统会自动识别你的职业、加载对应的战斗方案、检测副本数据。开箱即用。

**Q: 它会帮我走位吗？**

不会。HiAuRo 只负责按技能，不负责移动。走位和处理机制是你的工作。

**Q: 我可以暂停它吗？**

可以。你可以随时暂停（停止按技能但不关闭）或停止（完全关闭自动战斗）。

**Q: 不同副本的表现一样吗？**

不一样。有预设数据的高难度副本会使用执行轴/事实轴模式，表现更好。没有预设数据的普通副本会使用基础的无轴模式。

**Q: 我可以自己编辑战斗方案吗？**

可以。如果你是 ACR 作者，可以用 HiAuRo 提供的接口开发自己的职业战斗方案。普通用户也可以使用社区分享的方案。

## 技术细节（给想深入了解的人）

如果你对底层实现感兴趣：

- **每秒决策次数**：游戏 60 帧/秒，每帧都会执行一次完整的决策循环
- **技能优先级**：每个技能都有一个 Check 函数，返回正数表示"可以用"，返回负数表示"不能用"。系统按优先级从高到低检查
- **GCD/oGCD 分离**：GCD 技能（需要等公共冷却）和 oGCD 技能（不需要等）分开处理，确保不卡顿
- **目标选择**：支持多种策略（最近的敌人、血量最低的敌人、最佳 AOE 位置等）
- **热重载**：战斗方案可以热更新，不需要重启游戏
