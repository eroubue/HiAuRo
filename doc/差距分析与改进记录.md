### 动态减伤检查的实现细节

新增方法 `扫描已有减伤()`：
- 遍历 `Data.Party.All` 成员，检查每个成员的 `StatusList`
- 同时检查当前目标的 `StatusList`（覆盖雪仇等 debuff 类减伤）
- 使用注册时提供的 `状态ID` 字段匹配
- 状态ID 为 0 的技能跳过检测（总是重新分配）
- 使用 `HashSet<uint>` 去重，避免同一状态被多个职业重复计算

## 二、架构一致性分析

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 三轴同层级并行 | ✅ 一致 | AIRunner.Update() 中顺序调用，同层级 |
| 执行轴直接控制ACR | ✅ 一致 | PauseAcr/ForceSpell/ForceTarget 直接执行 |
| 事实轴经过智能层 | ✅ 一致 | 经 DecisionEngine + IntelligenceEngine |
| 辅助轴始终运行 | ✅ 一致 | 无条件调用，不受 ModeSwitch 控制 |
| 辅助轴向智能层提需求 | ⚠️ 设计目标 | 当前辅助轴直接 ForceSpell→SlotExecutor，理想路径是向 DemandBuffer 注册需求 |
| 智能层包含决策引擎+移动协调 | ✅ 功能正确 | 两者都在 AIRunner.UpdateFactAxis() 中被统一调度 |
